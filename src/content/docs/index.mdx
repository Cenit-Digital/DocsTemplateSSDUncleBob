El **Arnés SSD de Uncle Bob (edición artesano)** es un *harness*: el código, la
configuración y la lógica de ejecución que rodean al modelo para convertir una
conversación en software con disciplina. No añade otro modelo; añade **reglas,
orquestación y verificación** para que los agentes de IA trabajen como artesanos
y no como generadores de texto plausible.

## Qué resuelve

Los agentes escriben código con facilidad, pero sin restricciones producen
sistemas que nadie ha acordado y que nadie puede verificar. Este arnés impone un
único camino, gobernado y medible, desde la intención hasta el código probado:

1. **Spec conversada (SDD).** El agente `spec_partner` conversa y debate contigo
   hasta destilar un `project-spec.md` con decisiones razonadas.
2. **Gherkin.** El agente `gherkin_author` transforma esa spec en escenarios
   ejecutables `.feature`: el contrato del comportamiento.
3. **Puerta humana.** Apruebas el contrato Gherkin *antes* de que se escriba
   código. Es la única puerta de aprobación, y es la más barata.
4. **TDD estricto.** El agente `tdd_craftsman` implementa en ciclos
   rojo-verde-refactor, un test a la vez.
5. **Review y mutación.** El `judge` revisa disciplina y calidad; el
   `mutation_tester` mide si la suite realmente detecta cambios en el código.

:::tip[Documentación viva]
Este sitio no es un manual estático. Es **documentación viva** que se
auto-mejora: forma parte del arnés y evoluciona con el método a medida que el
propio sistema aprende de cada feature construida.
:::
